<section class="section">

    <div class="block-head">
      <span class="section-label">Why Prepo is needed</span>
      <h2>From a one-off delivery to lasting overview</h2>
    </div>
    <div class="prose prose--wide">
      <p>
        Many learning products start out neat: a first version, a clear link and a group of participants getting started.
        But over time, fragmentation often creeps in. New versions appear. Feedback is shared via email, documents or
        loose messages. Files get scattered. Test links keep circulating. Statistics aren't always easy to find. And when
        support is needed, it isn't always clear which version is in use.
      </p>
      <p>
        Prepo prevents management from becoming a loose collection of agreements, files and links. It brings structure
        around the learning experience: which products are active, which version is in use, where feedback arrives and how
        participants can reach the MicroGames or Stories.
      </p>
    </div>
  </div>
</section>
